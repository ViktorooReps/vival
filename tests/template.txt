Теперь формат тестов стал абсолютно свободным!
Полная анархия!

Парсер будет все игнорировать, кроме ключевых слов:
DESCRIPTION - описанние файла с тестами
COMMENT - комментарий для теста
INPUT - что подается программе на stdin
OUTPUT - что ожидается от программы на stdout
CMD - аргументы командной строки

Затем он берет последнее встретившееся ключевое
слово и соотносит его с содержимым в скобочках.

DESCRIPTION
/{Шаблон для файла с тестами}/

COMMENT
/{описание теста (не обязательно)}/
INPUT
/{что подается на stdin программе}/
OUTPUT
/{что ожидается на stdout}/

COMMENT
/{пример незаполненного теста}/
INPUT
/{все в скобочках можно разделять на несколько}/
/{частей - vival их соединит с помощью \n}/

COMMENT
/{а это просто пустой тест}/

PS: это не рабочий тест так как оказывается,
стандартный I/O на ASCII работает (какая неожиданность)
